# Deployment

Telysta Blog is deployed with GitHub Pages and GitHub Actions. Source code is pushed to GitHub; GitHub builds the site and publishes the generated `dist/` output.

## GitHub Pages Settings

In the GitHub repository:

1. Open `Settings`.
2. Open `Pages`.
3. Set `Source` to `GitHub Actions`.
4. After the first successful workflow run, set the custom domain to `telysta.com` if GitHub has not detected it automatically.
5. Enable `Enforce HTTPS` after DNS is verified.

## Build Settings

The workflow uses:

```txt
Node version: 22.12.0
Install: npm ci
Build command: npm run build
Output directory: dist
Production domain: https://telysta.com
```

`dist/` is generated by CI and must not be committed.

## Custom Domain

`public/CNAME` contains:

```txt
telysta.com
```

Astro copies this file into `dist/CNAME` during build, which lets GitHub Pages keep the custom domain.

## DNS

DNS records are configured at the domain DNS provider, not in this repository.

For the apex domain `telysta.com`, configure the records required by GitHub Pages for apex domains. If `www.telysta.com` is used later, configure it separately and decide whether it should redirect to `telysta.com`.

After DNS is configured, verify the domain in GitHub Pages settings and enable HTTPS.

## Local Workflow

Use local preview while editing:

```sh
npm run dev
```

Run a local production check when needed:

```sh
npm run build
```

Publishing happens after pushing source code to the `main` branch. GitHub Actions builds and deploys automatically.
