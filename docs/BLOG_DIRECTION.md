# 博客方向规划

## 核心定位

这个博客应该像一个安静、深色、带玻璃质感的个人档案馆。

主体风格接近 SHBlog Next：有氛围感、有空间层次、现代、简洁，适合中文写作。改进方向是降低视觉噪音、降低饱和度，让 ACG 气质藏在细节里，而不是直接变成显眼的主题。

这个站点不应该像一个高调的二次元主题站。它更应该像一个成熟的个人知识空间，只是在插画选择、图标、边角细节、微交互和氛围里，能慢慢看出一点 ACG 审美。

## 关键词

- 深色基底
- 玻璃拟态
- 空间层次
- 现代 UI
- 干净文字
- 轻 ACG 细节
- 低饱和
- 个人档案馆
- 写作优先

## 从 SHBlog Next 保留什么

- 深色氛围基础
- 空间感和层次感
- 玻璃面板质感
- 简洁的博客结构
- 对中文博客友好
- Astro 写作工作流
- 装饰性渐变
- 适合 MDX 的内容模型

## 需要改进什么

- 移除或弱化高饱和卡片。
- 避免大面积亮色块。
- 让卡片更安静、更统一。
- 让文章页更适合长时间阅读。
- UI 保持现代，但不要过度炫技。
- ACG 只作为隐藏气质，不作为中心主题。

## 视觉系统

整体配色应该是深色、冷调、低饱和。

建议色彩 token：

```txt
背景: #070A10 / #0B0F17
面板: rgba(18, 24, 36, 0.56)
面板悬停: rgba(25, 32, 46, 0.68)
边框: rgba(255, 255, 255, 0.08)
正文: #D8DEE9
弱化文字: #8B95A7
主强调色: #9CC7E8
柔和辅助色: #D6A7C8
```

设计原则：

> 颜色用来提示交互，不用来填满内容容器。

卡片、面板、标签和导航应主要依靠透明度、边框、模糊、阴影和留白建立质感，而不是依靠高饱和背景色。

## ACG 气质方向

ACG 气质应该是“靠近后才发现”，而不是第一眼就压过主体。

适合的方向：

- 低饱和的夜间房间插画
- 远景人物剪影
- 书桌、窗户、终端、书本、月光
- 类似动画 UI 的细线装饰
- 柔和的淡粉或月蓝高光
- 细节化的图标处理
- 在空状态、页脚、关于页里藏一点个人趣味

避免的方向：

- 高饱和角色 banner
- 每个页面都有大幅前景动漫角色
- 霓虹赛博风过载
- 过度可爱导致不稳重的 UI
- 亮色标签胶囊或彩色内容卡片

## Banner 策略

可以使用 banner，但它应该是氛围主导，而不是角色主导。

推荐方向：

> 安静的夜间场景，带玻璃、远处城市微光、书桌、终端、书本，以及非常轻微的 ACG 风格痕迹。如果有人物，也应该是环境的一部分，而不是画面的绝对主角。

首页 banner 负责建立情绪和空间感。关于页或 ACG 分类页可以放更明确一点的插画信号。

## 内容架构

文章可以按领域分散存放在多个 GitHub 仓库里，但应该由一个统一的 Astro 主站负责渲染和展示。

建议仓库拆分：

```txt
blog-site              # Astro 主站、主题、样式、部署
blog-notes-tech        # 技术笔记
blog-notes-life        # 随笔和个人写作
blog-notes-acg         # ACG、游戏、音乐、影像
blog-notes-archive     # 可选，长期归档
```

原则：

> 主站仓库负责呈现，内容仓库负责写作。

视觉系统、组件、路由和部署流程应该集中在 Astro 主站里，不要分散到每个内容仓库。

## 初始站点栏目

第一阶段先保持克制：

```txt
/          首页
/posts     全部文章
/archive   归档
/tags      标签
/about     关于
```

后续可扩展：

```txt
/notes     技术笔记
/essays    随笔
/media     ACG、游戏、音乐、影像
/now       近况
/links     友链和参考
```

## 部署方向

长期更推荐 Cloudflare，尤其是在使用自定义域名、预览部署、重定向、边缘能力或未来动态功能时。

如果只是做一个简单静态 MVP，GitHub Pages 也可以接受。

推荐方向：

```txt
GitHub 多仓库
+ GitHub Actions 聚合内容
+ Astro 构建
+ Cloudflare 部署
+ Cloudflare DNS / 自定义域名
```

## 实施阶段

### 第一阶段：MVP

- 基于 SHBlog Next 或类似 SHBlog Next 的 Astro 结构起步。
- 建立 Astro 项目和写作流程。
- 完成首页、文章列表、文章页、归档、标签、关于页。
- 配置 Markdown / MDX 内容。
- 添加 RSS 和 sitemap。
- 部署到 Cloudflare 或 GitHub Pages。

### 第二阶段：视觉打磨

- 替换全局颜色 token。
- 用低饱和玻璃质感重做卡片表面。
- 移除高饱和色块。
- 优化中文阅读排版。
- 重做标签、按钮、代码块和导航。
- 加入克制的 ACG 氛围细节。
- 检查桌面端和移动端布局。

### 第三阶段：内容系统

- 增加多内容仓库 checkout 工作流。
- 统一 frontmatter schema。
- 建立分类和标签策略。
- 视需要加入搜索。
- 如果长文系列变多，再加入系列文章支持。

### 第四阶段：身份层

- 加入氛围型 banner。
- 做一个更有个人气质的关于页。
- 可选加入 now 页面。
- 在页脚、空状态、元信息里加入安静的个人细节。
- 只有在确实需要互动时，再考虑评论系统。

## 设计准则

> 安静优先。玻璃其次。ACG 藏在细节里。写作永远是主角。
